AeroMed iOS Application

Overall Goal

We have been given the responsibility of creating an iOS 7 based application to be used by the AeroMed team at Spectrum Health. This application will be used to keep track of procedures and checklists of important vital tasks based on each procedure based on the patients conditions.

1 - Software

Our application is designed for the latest version of Apple's iOS 7 and will be developed within Apple's own Xcode IDE.

2 - Devices

The original project description state that the application would be for iPad's only, upon meeting with them further we will be making it for both iPad and iPhones running iOS 7 only.

3 - User Interface

The UI will be information driven to be able to look up a condition with a provided checklist of what is needed for the procedure.

Gantt Chart

Milestone One

We've had a handful goals set for the first milestone to have a working mock up and prototype of the UI to present.

Setup and Research

We spent some time in the first couple of weeks getting our project set up within Xcode, researching several frameworks after meeting with the client to integrate within our project to have a great looking and functional application.

Database

We chose to go with a NoSQL type database instead of a relational database, giving us the flexibility we desire. Parse was a great database with a unique and simple framework that works very nicely with iOS applications.

Design

Using the Pixate framework we have had the ability to use CSS stylesheets to design our iOS application in a unique and flexible way. It reduces the complexity of creating a really great looking app and offers a simpler alternative.

Prototypes

Currently there is no name for the application as the application may be used commercially not just internally.

Setbacks

Although we are on track, we did face a few setbacks. The following list desribes a few.

  • Objective-C learning curves
  • Scheduling Conflicts to Meet with Client
  • Awaiting Documents for Database

Milestone Two Preview

For the second milestone we have a few core functions we would like to get done to be able to show off in a functional prototype.

Checklist View

Implementing a checklist view is an essential core functionality to the application so we would like to start there to integrate an easy to view checklist populated based on the procedure pulled from the database.

Document Viewer

By creating the document viewer we would be able to pull in all of the other information for each procedure pulled from the database other than the checklist. Examples would be common symptoms, resources, alternatives, etc.

Performance Charts

The core function of the admin users in the application would to monitor perofrmance trends of the core users checking items off the list. We would like to present the trends in charts with simple animations that do not take away from the importance of the information at hand.
